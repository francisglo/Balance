🔐 DEPLOYMENT PROTECTION - EXPLICACIÓN
═════════════════════════════════════════════════════════════════════════════

¿Qué es?
   Por defecto, Vercel habilita SSO (Single Sign-On) que requiere autenticación
   para acceder a cualquier función o endpoint.

¿Por qué se necesita deshabilitar?
   Sin deshabilitar, TODOS los requests a /think/analyze, /health, etc.
   retornarán 401 Unauthorized (SSO wall).

¿Es seguro deshabilitar?
   SÍ - THINK API no tiene datos sensibles (análisis de reuniones)
   Si necesitas proteger endpoints específicos, habilita auth en código.

❓ FAQ RÁPIDO
═════════════════════════════════════════════════════════════════════════════

P: "¿Qué pasa si no deshabilito Deployment Protection?"
R: El API no será accesible públicamente (retorna 401 para todo)
   La web cargaré pero no podrá llamar al API.

P: "¿Necesito PostgreSQL para que funcione?"
R: NO - El API tiene fallback a in-memory. Funciona sin DB.
   Si necesitas PERSISTENCIA, configura PostgreSQL en Vercel.

P: "¿Cuáles son los credentials de LiveKit?"
R: LiveKit es OPCIONAL. Sin credentials, el button de video no funciona
   pero el Think Engine SÍ funciona (análisis de reuniones).

P: "¿Cómo agrego credenciales LiveKit?"
R: 1. Crea cuenta en https://livekit.io
   2. Obtén API_KEY, API_SECRET, URL
   3. Agrega en Vercel Settings → Environment Variables
   4. Vercel redepliega automáticamente

P: "¿Cómo vuelvo a deployar después de cambios?"
R: cd apps/api && vercel deploy --prod
   cd ../web && vercel deploy --prod


🆘 TROUBLESHOOTING
═════════════════════════════════════════════════════════════════════════════

PROBLEMA: "API retorna 401 Unauthorized"
─────────────────────────────────────────
Causa: Deployment Protection aún activo
Solución: Ejecuta Disable-DeploymentProtection.ps1

PROBLEMA: "Test-Endpoints.ps1 falla"
─────────────────────────────────────
Causa: Probablemente Deployment Protection activo
Solución: Espera 60+ segs después de deshabilitar, retry

PROBLEMA: "LiveKit token es null"
─────────────────────────────────
Causa: NORMAL - sin credenciales LiveKit configuradas
Solución: Agregar credenciales en Vercel Settings (opcional)

PROBLEMA: "Reuniones no se guardan"
─────────────────────────────────────
Causa: Sin PostgreSQL en producción
Solución: Configura DATABASE_URL en Vercel Settings
